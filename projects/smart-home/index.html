Case Study Details
Multi-node BLE Sensor Mesh ESP32ArduinoBLEC++SensorsRelay Control

A decentralized automation system running local relay boards and sensor nodes that connect with a central dashboard via BLE and WiFi, prioritizing private automation.

System Architecture

Sensors (Temp/Humidity/PIR) -> ESP32 Node -> BLE Advertisement -> Central Hub -> Automatic rules / UI update.

Technical Challenges

Power consumption of BLE sensor nodes. Addressed by configuring ESP32 deep-sleep modes, waking nodes up every 5 minutes to transmit data before entering sleep states.

Results & Impact

Deployed local multi-node sensors across rooms. Features automatic air-quality monitoring and relay power switches, accessible via a responsive dashboard running inside the local network.

Lessons Learned

Developed expertise in battery optimization, sensor data noise filters, and local network protocols.
