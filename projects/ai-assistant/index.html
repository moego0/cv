Case Study Details

An interactive desktop client using speech recognition, text-to-speech, and vision models to build a custom desktop dashboard and control physical smart home modules.

System Architecture

Microphone Audio -> Local Web Speech / Python SpeechSTT -> Text parsing -> LLM Reasoning (Gemini) -> Action triggering (App execution / BLE command) -> Speak reply.

Technical Challenges

Achieving low-latency voice command responses in Arabic. Addressed by utilizing dual speech recognizers and preprocessing common command tags locally to bypass network checks.

Results & Impact

Completed a functional UI with audio visualizer waveform. The assistant executes shell commands, transcribes voice in real-time, and handles basic vision tasks like describing objects placed in front of the camera.

Lessons Learned

Mastered thread synchronization in Qt applications, BLE GATT services, and modular integration of vision APIs.
